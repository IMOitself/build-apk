## Commit Message Guidelines

to maintain a clean and readable history, please prefix your commit messages using the following categories:

| prefix | description |
| --- | --- |
| **`feat:`** | changes that are visible to the user within the app. |
| **`fix:`** | fix a bug. |
| **`refactor:`** | **not** visible to the user i.e internal code changes, logic improvements, or cleanups. |
| **`readme:`** | updates specifically to `README.md` or assets/files directly referenced within the README. |
| **`repo:`** | changes to files for working on this project i.e build settings (`build.gradle`) or file updates from original CodeAssist. |

### Examples

* `feat: add dark mode toggle to settings screen`
* `fix: resolve crash on app launch`
* `refactor: extract string constants to separate file`
* `readme: update installation instructions`
* `repo: update gradle version`
